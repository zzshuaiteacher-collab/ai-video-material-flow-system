# 快速开始

本文件是总控 Agent 可发送给客户的用户版快速说明；客户直接看到仓库时，也可以用它了解如何启动。

你不需要先理解所有文件，只要按下面步骤做，就能启动一个素材流自动化剪辑系统的搭建过程。

## 1. 准备一个总控对话框

打开 Codex，新建一个对话框，把它作为：

```text
总控 Agent
```

总控 Agent 的任务不是马上剪视频，而是先带你搭系统。

本项目默认搭配飞书多维表格 + Codex 使用：飞书负责承载任务表、素材表和执行记录表，Codex 负责总控协作、专项 Agent 执行、文档更新和复盘沉淀。

## 2. 让总控读取仓库入口

直接告诉它：

```text
请读取本仓库的 README.md，作为总控 Agent 主动引导我搭建素材流自动化剪辑系统。
```

`README.md` 会告诉总控下一步应读取哪些文件、先向你说明什么，以及如何推进飞书和专项 Agent 的配置。

## 3. 回答总控 Agent 的启动问题

总控 Agent 会先问你：

1. 这个项目服务什么品牌、账号、产品或内容主题？
2. 你的素材目录在哪里？
3. 有没有参考素材或已有剪辑模板？
4. 最终发布平台是什么？
5. 当前是否已经安装/授权飞书 CLI，或者是否已经有飞书多维表格？

如果你还没有完整答案，可以先给大概信息。

## 4. 补全项目说明文档

总控 Agent 会维护：

```text
docs/项目说明文档.md
```

这份文档是整个项目的真源文档。

后续项目规则、目录、协作方式、表格字段、复盘结论都应该更新到这里。

## 5. 先搭协作系统

第一轮先不要直接剪片。总控会先引导你完成：

```text
安装/授权飞书 CLI
→ 创建 3 张飞书协作表
→ 创建素材、内容生产 2 个专项 Agent
→ 给第一条视频建任务记录
```

总控首次回复时应同时向你发送：

```text
docs/首次协作说明.md
```

## 6. 用第一条视频跑完整 MVP

系统搭好后，再跑通：

```text
素材
→ 参考素材分析与写作标准
→ 任务单
→ 视频初稿
→ 内容生产合规自检
→ 用户人工终审与发布决定
→ 归档
→ 复盘
```

这个叫：

```text
流程闭环
```

协作表应在第一条视频开始前已经建立：

1. 视频任务总表
2. 素材库表
3. Agent 执行记录表

字段参考：

```text
docs/协作表字段设计.md
```

如果你使用飞书，参考：

```text
docs/飞书多维表格搭建指南.md
```

## 7. 专项 Agent

默认只有 3 个 Agent：

1. 总控 Agent
2. 素材 Agent
3. 内容生产 Agent

审核、发布决定由用户完成，总控负责记录状态、归档与复盘。

启动 Prompt 在：

```text
templates/agent-prompts/
```

## 8. 每轮结束都复盘

复盘时不要只问“视频好不好看”，还要问：

1. 本轮是否闭环？
2. 哪一步返工最多？
3. 哪些字段不够？
4. 哪些流程可以标准化？
5. 哪些流程适合沉淀成 skill？

复盘清单在：

```text
templates/checklists/总控复盘清单.md
```

## 9. 最重要的边界

不要把“流程闭环”说成“真实效果闭环”。

只有完成：

```text
真实发布
→ 数据回填
→ 效果分析
→ 模板优化
```

才算效果闭环。

## 10. 推荐第一次使用路径

```text
README.md（总控入口）
→ 总控项目引导.md（详细手册）
→ docs/首次协作说明.md
→ docs/项目说明文档.md
→ docs/飞书多维表格搭建指南.md
→ templates/agent-prompts/
→ docs/MVP协作执行工作流模板.md
→ templates/tables/
→ templates/checklists/
```
