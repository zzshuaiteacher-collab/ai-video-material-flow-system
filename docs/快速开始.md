# 快速开始

本文件给第一次使用本项目的人看。

你不需要先理解所有文件，只要按下面步骤做，就能启动一个素材流自动化剪辑系统的搭建过程。

## 1. 准备一个总控对话框

打开你常用的 AI 编程或协作工具，新建一个对话框，把它作为：

```text
总控 Agent
```

总控 Agent 的任务不是马上剪视频，而是先带你搭系统。

## 2. 把总控引导发给它

把这个文件发给总控 Agent：

```text
总控项目引导.md
```

或者直接告诉它：

```text
请读取本仓库的 README.md、AGENT.md、总控项目引导.md 和 docs/项目说明文档.md，作为总控 Agent 引导我搭建素材流自动化剪辑系统。
```

## 3. 回答总控 Agent 的启动问题

总控 Agent 会先问你：

1. 这个项目服务什么品牌、账号、产品或内容主题？
2. 你的素材目录在哪里？
3. 有没有参考视频或剪辑模板？
4. 最终发布平台是什么？
5. 本轮目标是本地 MVP，还是直接接入协作表？

如果你还没有完整答案，可以先给大概信息。

## 4. 补全项目说明文档

总控 Agent 会维护：

```text
docs/项目说明文档.md
```

这份文档是整个项目的真源文档。

后续项目规则、目录、协作方式、表格字段、复盘结论都应该更新到这里。

## 5. 先跑本地 MVP

第一轮不要急着创建很多 Agent，也不要急着接入复杂数据库。

先跑通：

```text
素材
→ 任务单
→ 视频初稿
→ AI 初审
→ 人工终审
→ 归档
→ 复盘
```

这个叫：

```text
流程闭环
```

## 6. 再接入协作表

本地 MVP 跑通后，再建立 3 张协作表：

1. 视频任务总表
2. 素材与模板表
3. Agent 执行记录表

字段参考：

```text
docs/协作表字段设计.md
```

如果你使用飞书，参考：

```text
docs/飞书多维表格搭建指南.md
```

## 7. 再拆分多 Agent

默认只拆 4 个 Agent：

1. 总控 Agent
2. 素材与模板 Agent
3. 内容生产 Agent
4. 审核发布 Agent

启动 Prompt 在：

```text
templates/agent-prompts/
```

## 8. 每轮结束都复盘

复盘时不要只问“视频好不好看”，还要问：

1. 本轮是否闭环？
2. 哪一步返工最多？
3. 哪些字段不够？
4. 哪些流程可以标准化？
5. 哪些流程适合沉淀成 skill？

复盘清单在：

```text
templates/checklists/总控复盘清单.md
```

## 9. 最重要的边界

不要把“流程闭环”说成“真实效果闭环”。

只有完成：

```text
真实发布
→ 数据回填
→ 效果分析
→ 模板优化
```

才算效果闭环。

## 10. 推荐第一次使用路径

```text
README.md
→ 总控项目引导.md
→ docs/项目说明文档.md
→ docs/MVP协作执行工作流模板.md
→ templates/tables/
→ templates/agent-prompts/
→ templates/checklists/
```
