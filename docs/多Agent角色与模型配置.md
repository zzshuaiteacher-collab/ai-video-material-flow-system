# 多 Agent 角色与模型配置

## 1. 默认 Agent

MVP 阶段建议只使用 4 个 Agent。

| Agent | 默认职责 | 什么时候创建 |
| --- | --- | --- |
| 总控 Agent | 项目引导、派单、验收、文档、复盘、skill 判断 | 一开始就创建 |
| 素材与模板 Agent | 素材整理、命名、参考拆解、模板维护 | 本地 MVP 稳定后创建 |
| 内容生产 Agent | 任务单、字幕、视频初稿、生成记录 | 素材和模板清楚后创建 |
| 审核发布 Agent | AI 初审、人工终审协助、发布归档、数据回填 | 内容生产开始后创建 |

## 2. 不建议一开始创建的 Agent

MVP 阶段不建议单独创建：

- Skill 沉淀 Agent
- 数据分析 Agent
- 发布 Agent
- 项目经理 Agent
- 质量检查 Agent

原因：这些职责可以先由 4 个核心 Agent 兼任，等任务量增加后再拆。

## 3. 模型选择原则

模型名称不绑定任何供应商。用户可以根据自己使用的平台替换。

| 工作类型 | 推荐模型等级 |
| --- | --- |
| 总控方案、冲突判断、复盘、skill 是否沉淀 | 强模型 |
| 首条视频模板、创意方向、复杂内容取舍 | 中高模型 |
| 普通素材整理、批量命名、字段回填 | 中低成本模型 |
| 明确脚本执行、批量生成版本 | 代码/执行型模型 |
| 合规争议、广告法、肖像、隐私判断 | 强模型 + 人工确认 |

## 4. Agent 推荐配置

| Agent | 默认模型 | 可降级场景 | 需升级场景 |
| --- | --- | --- | --- |
| 总控 Agent | 强模型 | 只查状态、整理格式 | 系统设计、流程冲突、复盘 |
| 素材与模板 Agent | 中模型 | 批量重命名、台账整理 | 分类争议、模板定型 |
| 内容生产 Agent | 中高模型 | 按明确任务单执行 | 首条模板、创意判断、字幕风格 |
| 审核发布 Agent | 中模型 | 归档和字段回填 | 合规争议、隐私风险、版权风险 |

## 5. 升级规则

如果 Agent 认为当前模型无法完成判断，不要硬做。

它应该在执行记录中写：

```text
当前任务难度升级，原因是：
1. ...
2. ...
建议由总控决定是否切换更强模型。
```

## 6. 成本控制原则

1. 强模型用在判断，不用在机械搬运。
2. 低成本模型用在批量整理，不用在复杂合规。
3. 视频生成前先任务单确认，减少反复渲染成本。
4. 每条视频都记录使用模型，便于后续测算 ROI。
