# 安装与运行

本项目不是一个开箱即跑的视频渲染程序，而是一套素材流自动化剪辑协作系统模板。

它的核心交付物是：

- 文档
- Agent Prompt
- 协作表字段
- 审核清单
- skill 蓝图

因此，第一次使用不需要安装视频渲染引擎，但默认需要配置 Codex 和飞书 CLI。

## 1. 获取项目

方式一：直接下载 GitHub ZIP。

方式二：使用 Git：

```bash
git clone https://github.com/zzshuaiteacher-collab/ai-video-material-flow-system.git
cd ai-video-material-flow-system
```

## 2. 推荐运行环境

默认推荐环境：

1. Codex：用于创建总控 Agent 和专项 Agent
2. 飞书多维表格：用于创建视频任务总表、素材库表、Agent 执行记录表
3. 一个素材目录
4. 一个输出目录

如果要迁移到其他工具，你需要至少准备：

1. 一个支持读取项目文件的 AI 工具或 Agent 环境
2. 一个素材目录
3. 一个输出目录
4. 一个协作表工具

可选协作表工具：

- 飞书多维表格
- Notion
- Airtable
- Google Sheet
- Excel

## 3. 安装并检查飞书 CLI

总控 Agent 在创建飞书表格前先自检：

```powershell
Get-Command lark-cli -ErrorAction SilentlyContinue
lark-cli --version
```

如果未安装，在已经安装 Node.js 和 `npm` 的环境中执行：

```powershell
npm install -g @larksuite/cli
```

包地址：

```text
https://www.npmjs.com/package/@larksuite/cli
```

安装完成后重新执行 `lark-cli --version` 确认可用。总控不能在 CLI 未安装或未获得飞书授权时声称已经完成建表。

## 4. 非阻塞授权流程

`lark-cli config init --new` 和直接运行的 `lark-cli auth login` 都可能等待用户打开浏览器完成授权，Agent 不应把它们当作普通同步命令直接卡在当前步骤。

推荐流程：

1. 总控在后台启动 `lark-cli config init --new`，取得返回的授权链接并原样发给用户。
2. 用户完成应用配置后，总控发起非阻塞用户授权：

```powershell
lark-cli auth login --recommend --no-wait --json
```

3. 总控把返回的 `verification_url` 原样发给用户，并保存 `device_code`。
4. 用户确认浏览器授权完成后，总控在下一步收尾：

```powershell
lark-cli auth login --device-code <device_code>
```

不要依赖二维码生成作为必需步骤；把 `verification_url` 直接交给用户是最稳定的路径。

## 5. 不需要立即安装的东西

第一轮不需要：

- 视频渲染引擎
- 自动发布工具
- 数据抓取工具
- 数据库
- 服务器

先把协作表和 Agent 搭起来，再用第一条视频跑通流程。

## 6. Agent 启动方式

新建一个总控对话框，把以下内容给它：

```text
请读取 README.md、AGENT.md、总控项目引导.md、docs/项目说明文档.md，并作为总控 Agent 引导我搭建素材流自动化剪辑系统。
```

如果你的 Agent 工具不能直接读取文件，就把 `总控项目引导.md` 的全文复制给它。

## 7. 建立目录

建议在你的具体项目里建立：

```text
materials/
outputs/
docs/
templates/
tools/
```

真实素材默认不要提交到 GitHub。

## 8. 建立协作表

如果你使用飞书：

1. 新建一个多维表格
2. 按 `docs/飞书多维表格搭建指南.md` 建 3 张表
3. 导入或复制 `templates/tables/` 里的表头
4. 建立总控看板和各 Agent 待处理视图

如果你不用飞书，也可以按 `docs/协作表字段设计.md` 在其他工具里创建。

## 9. 使用 skill 蓝图

skill 蓝图在：

```text
skills/blueprints/
```

如果你的平台支持 skill，可以把里面的 `SKILL.md` 安装到对应位置。

如果不支持，也可以把它们当作专项工作流文档，让 Agent 读取。

## 10. 常见问题

### 这个项目能直接生成视频吗？

不能。

本仓库是系统模板，不绑定某个视频生成引擎。你可以后续接入 ffmpeg、Remotion、剪映自动化、CapCut 或其他工具。

### 一开始要不要建很多 Agent？

只建最小的 4 个角色。

总控先建立飞书三张表和素材、内容生产两个专项 Agent，再使用第一条视频测试这套完整协作链路。

### 一开始要不要建很多表？

不要。

先建 3 张表：视频任务总表、素材库表、Agent 执行记录表。

### 能不能直接公开上传客户素材？

不建议。

先看：

```text
docs/公开发布与隐私检查.md
```
