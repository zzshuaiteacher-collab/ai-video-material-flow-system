# AI 素材流自动化剪辑协作系统模板

这是一个用于搭建“短视频素材流自动化剪辑系统”的通用项目模板。

它不是某一个品牌的定制项目，而是一套可迁移的协作方法：用户只需要准备素材、模板、协作表和一个总控 Agent，就可以逐步搭建自己的内容生产系统。

## 适用场景

- 餐饮门店短视频
- 探店素材整理与剪辑
- 产品展示短视频
- 本地生活账号素材流
- 课程、口播、服务类内容的批量剪辑
- 任何需要“素材整理 → 内容生成 → 审核 → 发布归档 → 复盘”的项目

## 项目核心思想

先跑通一个最小闭环，再拆分多 Agent。

推荐顺序：

```text
总控 Agent 初始化项目
→ 本地整理素材和规则
→ 跑通一条 MVP 视频
→ 建立协作表
→ 拆分多 Agent
→ 再跑一条多 Agent MVP
→ 真实发布与数据回填
→ 复盘并沉淀 skill
```

## 角色设计

默认只需要 4 个 Agent：

| Agent | 职责 |
| --- | --- |
| 总控 Agent | 引导用户搭建系统、派单、检查状态、维护文档、复盘、判断是否沉淀 skill |
| 素材与模板 Agent | 整理素材、分类命名、读取参考视频、维护素材与模板状态 |
| 内容生产 Agent | 生成剪辑任务单、选择素材和音乐、生成视频初稿、输出生成记录 |
| 审核发布 Agent | AI 合规初审、协助人工终审、发布归档、提醒数据回填 |

不建议一开始创建很多 Agent。系统越早越复杂，越难跑通。

## Skill 蓝图

本项目包含 3 个通用 skill 蓝图：

| Skill | 用途 |
| --- | --- |
| `material-flow-organizer` | 素材扫描、命名、分类、首次整理、增量整理 |
| `reference-video-decomposer` | 参考视频拆解、节奏分析、模板提炼 |
| `short-video-compliance-guard` | 内容生产前置合规、AI 初审、人工终审 |

位置：

```text
skills/blueprints/
```

这些是通用脱敏版，可以按用户自己的 Agent 平台安装或改写。

## 快速开始

1. 新建一个总控对话框。
2. 把 [总控项目引导.md](./总控项目引导.md) 发给总控 Agent。
3. 让总控 Agent 按阶段向你提问，并生成项目说明文档。
4. 准备你的素材目录和参考视频目录。
5. 按 [docs/MVP协作执行工作流模板.md](./docs/MVP协作执行工作流模板.md) 跑第一条 MVP。
6. 当本地 MVP 跑通后，再接入协作表和多 Agent。

如果你使用飞书多维表格，请参考：

[docs/飞书多维表格搭建指南.md](./docs/飞书多维表格搭建指南.md)

如果想先了解每个文件的用途，阅读：

[项目包清单.md](./项目包清单.md)

## 当前模板验证状态

本模板来自一个真实素材流项目的 MVP 抽象。

已经验证：

- 单总控本地 MVP 可以跑通。
- 4 Agent + 3 协作表的简化架构可以跑通。
- “先任务单，后生成视频”的内容生产方式可以降低返工。
- “AI 初审 + 人工终审”的审核机制可以形成清晰责任边界。

尚未包含：

- 开箱即用的视频渲染引擎。
- 自动发布到平台。
- 自动抓取平台数据。
- 通用商业授权音乐方案。

## 重要提醒

- 不要把未授权素材、顾客正脸、订单、小票、支付码、手机号上传到公开仓库。
- 不要把飞书、Notion、Airtable、Google Sheet 等协作平台 token 上传到仓库。
- 不要把“流程闭环验证”夸大成“真实投放效果验证”。
- 真实发布前，必须由人工确认版权、肖像、价格、地址、活动、广告法等风险。

公开发布前请阅读：

[docs/公开发布与隐私检查.md](./docs/公开发布与隐私检查.md)
