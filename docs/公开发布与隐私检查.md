# 公开发布与隐私检查

本文件用于在项目上传 GitHub、网站、公开文档或交给第三方前做检查。

## 1. 不要上传的内容

不要上传：

- 平台 token
- API key
- Cookie
- 账号密码
- 私密飞书、Notion、Airtable、Google Sheet 链接
- 客户合同或报价
- 未授权顾客正脸
- 订单、小票、手机号、地址、支付码
- 未授权音乐
- 未授权达人视频或参考视频原文件
- 原始客户素材，除非确认可公开

## 2. 推荐公开的内容

可以公开：

- 通用项目说明
- 通用工作流
- 通用表格字段
- Agent Prompt 模板
- 审核清单
- 复盘示例
- 不含真实客户信息的示例数据

## 3. 文案口径

如果只跑通流程，应写：

```text
已完成流程闭环验证。
```

不要写：

```text
已完成真实投放验证。
已验证 ROI。
已实现全自动发布。
```

除非确实完成了真实发布、数据回填和效果复盘。

## 4. 发布链接类型

任何复盘中出现链接时，都要标记类型：

| 类型 | 说明 |
| --- | --- |
| 真实发布 | 真实账号发布，可用于效果复盘 |
| 占位测试 | 只用于流程测试，不可用于效果判断 |
| 竞品参考 | 只用于拆解参考，不算项目发布 |

## 5. GitHub 前检查清单

- [ ] `.env` 已排除
- [ ] token、key、cookie 已排除
- [ ] 原始客户素材已排除
- [ ] 真实顾客和订单信息已排除
- [ ] 未授权音乐和参考视频已排除
- [ ] 本机绝对路径已替换为占位路径
- [ ] 品牌定制信息已抽象为通用说明
- [ ] 文案没有夸大成真实效果验证
- [ ] `.gitignore` 已覆盖临时文件和隐私素材目录

## 6. 总控 Agent 发布前提醒

在用户要求上传前，总控 Agent 应提醒：

```text
我建议先做一次公开发布检查，确认没有 token、私密链接、真实客户素材、顾客隐私或未授权素材。
同时确认当前项目对外描述为“流程闭环验证”，不要写成“真实投放效果验证”。
```
