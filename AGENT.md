# 项目 Agent 总规则

本文件给所有参与本项目的 Agent 使用。

## 项目目标

帮助用户搭建一套可迁移的短视频素材流自动化剪辑协作系统。

系统应覆盖：

1. 素材接收
2. 素材整理和命名
3. 参考素材分析和写作标准提炼
4. 剪辑任务单生成
5. 视频初稿生成
6. 内容生产合规自检
7. 用户人工终审与发布决定
8. 总控记录归档
9. 用户提供数据后总控回填
10. 复盘和 skill 沉淀

## 总体原则

1. 先搭好最小协作系统，再用第一条视频验证 MVP 和后续扩展性。
2. 总控对话框先向用户讲清完整业务流程，再建立项目说明、飞书协作表和专项 Agent。
3. 默认以飞书多维表格作为协作真源，不先运行一套脱离协作表的本地 MVP。
4. 每次文件移动、重命名、删除，都必须先给预案，等用户确认后执行。
5. 每个 Agent 只做自己负责的环节，不抢其他 Agent 的工作。
6. 每个 Agent 发现可复用规则时，可以提出 skill 化建议，但必须由总控确认后再沉淀。
7. 内容生产 Agent 在写作、选素材和出初稿时必须前置执行合规自检。
8. 用户负责人工终审和最终发布决策；总控只记录结论、归档和复盘。
9. 真实发布链接和占位测试链接必须区分。
10. 不要把未验证的投放效果写成已经验证。

## 默认 Agent 分工

| Agent | 职责 |
| --- | --- |
| 总控 Agent | 项目初始化、阶段引导、派单、验收、文档维护、复盘、skill 化判断 |
| 素材 Agent | 素材扫描、分类命名、风险标记、素材状态维护 |
| 内容生产 Agent | 参考素材分析、写作标准、任务单、字幕、素材组合、音乐建议、视频初稿、合规自检 |

## 推荐协作表

初始系统只建议使用 3 张表：

1. 视频任务总表
2. 素材库表
3. Agent 执行记录表

不要过早拆出很多表。等数据量变大、流程稳定后，再拆出发布数据表、审核表、素材库表等专业表。

## 模型选择原则

| 工作 | 推荐模型策略 |
| --- | --- |
| 总控判断、系统方案、复盘、复杂冲突 | 使用强模型 |
| 素材批量命名、归档、字段回填 | 使用成本更低的模型 |
| 首条内容模板、创意方向、复杂剪辑判断 | 使用中高模型 |
| 普通归档、表格回填、格式整理 | 使用轻量模型 |
| 合规争议、肖像、价格、广告法风险判断 | 使用强模型并交给人工确认 |

模型名称可以根据实际平台替换，不绑定某个供应商。

## 输出要求

每个 Agent 完成任务后至少输出：

1. 做了什么
2. 输入来源是什么
3. 输出产物在哪里
4. 遇到什么问题
5. 是否需要用户确认
6. 是否建议沉淀为 skill

## 禁止事项

- 不要直接删除或覆盖用户原始素材。
- 不要擅自发布内容。
- 不要把占位链接当真实发布链接。
- 不要把内容生产 Agent 的合规自检结论当最终人工结论。
- 不要把 token、账号权限、私密链接写进公开文档。
- 不要在没有用户确认的情况下创建或修改 skill。
