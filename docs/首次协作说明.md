# 首次协作说明

这份简版说明由总控 Agent 在第一次回复用户时发送，用来让用户先知道整个系统如何搭建、谁负责什么，以及什么时候需要用户确认。

## 我们将如何协作

```text
确认项目与素材信息
→ 建立项目说明文档
→ 安装并授权飞书 CLI
→ 创建飞书三张协作表
→ 创建素材和内容生产两个专项 Agent
→ 素材整理与模板确认
→ 参考素材分析、写作标准与视频初稿
→ 内容合规自检和用户人工终审
→ 用户发布决定、总控归档与复盘
```

## 三个 Agent 与用户职责

| Agent | 负责什么 |
| --- | --- |
| 总控 Agent | 搭建系统、派单、验收、维护文档、复盘 |
| 素材 Agent | 只整理素材：扫描、命名、分类、风险标记和素材状态维护 |
| 内容生产 Agent | 分析参考素材、提炼写作标准、生成任务单和初稿、做合规自检 |
| 用户 | 人工终审并决定是否发布；向总控提供最终发布状态与数据 |

## 三张飞书表

| 表 | 用途 |
| --- | --- |
| 视频任务总表 | 记录每条视频从选题到归档和复盘的状态 |
| 素材库表 | 记录原始/整理后素材、音乐与素材风险；不承载参考分析或写作标准 |
| Agent 执行记录表 | 记录 Agent 做了什么、产出了什么、遇到什么问题 |

## 用户需要确认的事情

1. 项目目标、素材目录、发布平台和第一条视频方向。
2. 飞书授权或协作表创建所需操作。
3. 素材移动和重命名预案。
4. 参考分析结论、写作标准、剪辑任务单与需要使用的素材、音乐。
5. 合规风险和最终是否发布。
6. 哪些成熟流程值得沉淀为 skill。

## 第一轮的完成标准

第一轮不是“先在本地随便跑一条”，而是在搭好的飞书协作与 Agent 分工中跑通一条完整任务：

```text
素材整理 → 参考分析与任务单 → 视频初稿与合规自检 → 用户人工终审/发布决定 → 总控归档 → 复盘
```

完成这个流程，代表系统具备协作运行能力；只有真实发布并回填数据后，才能判断真实内容效果。
