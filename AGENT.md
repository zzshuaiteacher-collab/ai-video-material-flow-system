# 项目 Agent 总规则

本文件给所有参与本项目的 Agent 使用。

## 项目目标

帮助用户搭建一套可迁移的短视频素材流自动化剪辑协作系统。

系统应覆盖：

1. 素材接收
2. 素材整理和命名
3. 参考视频拆解
4. 剪辑任务单生成
5. 视频初稿生成
6. AI 合规初审
7. 人工终审
8. 发布归档
9. 数据回填
10. 复盘和 skill 沉淀

## 总体原则

1. 先跑通 MVP，再追求自动化和规模化。
2. 先在一个总控对话框内跑通，再拆分多 Agent。
3. 先本地验证流程和字段，再迁移到协作表。
4. 每次文件移动、重命名、删除，都必须先给预案，等用户确认后执行。
5. 每个 Agent 只做自己负责的环节，不抢其他 Agent 的工作。
6. 每个 Agent 发现可复用规则时，可以提出 skill 化建议，但必须由总控确认后再沉淀。
7. 合规审核要前置到内容生产阶段，不要只放在最后。
8. AI 初审负责提示风险，人工终审负责最终发布决策。
9. 真实发布链接和占位测试链接必须区分。
10. 不要把未验证的投放效果写成已经验证。

## 默认 Agent 分工

| Agent | 职责 |
| --- | --- |
| 总控 Agent | 项目初始化、阶段引导、派单、验收、文档维护、复盘、skill 化判断 |
| 素材与模板 Agent | 素材扫描、分类命名、参考视频拆解、模板和素材状态维护 |
| 内容生产 Agent | 任务单、字幕、素材组合、音乐建议、视频生成或剪辑指令 |
| 审核发布 Agent | AI 初审、人工终审清单、发布归档、数据回填提醒 |

## 推荐协作表

MVP 阶段只建议使用 3 张表：

1. 视频任务总表
2. 素材与模板表
3. Agent 执行记录表

不要过早拆出很多表。等数据量变大、流程稳定后，再拆出发布数据表、审核表、素材库表等专业表。

## 模型选择原则

| 工作 | 推荐模型策略 |
| --- | --- |
| 总控判断、系统方案、复盘、复杂冲突 | 使用强模型 |
| 素材批量命名、归档、字段回填 | 使用成本更低的模型 |
| 首条内容模板、创意方向、复杂剪辑判断 | 使用中高模型 |
| 普通归档、表格回填、格式整理 | 使用轻量模型 |
| 合规争议、肖像、价格、广告法风险判断 | 使用强模型并交给人工确认 |

模型名称可以根据实际平台替换，不绑定某个供应商。

## 输出要求

每个 Agent 完成任务后至少输出：

1. 做了什么
2. 输入来源是什么
3. 输出产物在哪里
4. 遇到什么问题
5. 是否需要用户确认
6. 是否建议沉淀为 skill

## 禁止事项

- 不要直接删除或覆盖用户原始素材。
- 不要擅自发布内容。
- 不要把占位链接当真实发布链接。
- 不要把 AI 初审结论当最终人工结论。
- 不要把 token、账号权限、私密链接写进公开文档。
- 不要在没有用户确认的情况下创建或修改 skill。
