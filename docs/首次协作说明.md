# 首次协作说明

这份简版说明由总控 Agent 在第一次回复用户时发送，用来让用户先知道整个系统如何搭建、谁负责什么，以及什么时候需要用户确认。

## 我们将如何协作

```text
确认项目与素材信息
→ 建立项目说明文档
→ 安装并授权飞书 CLI
→ 创建飞书三张协作表
→ 创建三个专项 Agent
→ 素材整理与模板确认
→ 内容生产与视频初稿
→ AI 初审和人工终审
→ 发布归档与复盘
```

## 四个 Agent

| Agent | 负责什么 |
| --- | --- |
| 总控 Agent | 搭建系统、派单、验收、维护文档、复盘 |
| 素材与模板 Agent | 整理素材、命名分类、拆解参考、维护模板 |
| 内容生产 Agent | 生成任务单、字幕、素材组合和视频初稿 |
| 审核发布 Agent | AI 初审、协助人工终审、发布归档和数据回填提醒 |

## 三张飞书表

| 表 | 用途 |
| --- | --- |
| 视频任务总表 | 记录每条视频从选题到归档和复盘的状态 |
| 素材与模板表 | 记录素材、音乐、参考视频、模板与风险 |
| Agent 执行记录表 | 记录 Agent 做了什么、产出了什么、遇到什么问题 |

## 用户需要确认的事情

1. 项目目标、素材目录、发布平台和第一条视频方向。
2. 飞书授权或协作表创建所需操作。
3. 素材移动和重命名预案。
4. 剪辑任务单与需要使用的素材、音乐。
5. 合规风险和最终是否发布。
6. 哪些成熟流程值得沉淀为 skill。

## 第一轮的完成标准

第一轮不是“先在本地随便跑一条”，而是在搭好的飞书协作与 Agent 分工中跑通一条完整任务：

```text
素材 → 任务单 → 视频初稿 → AI 初审 → 人工终审 → 归档 → 复盘
```

完成这个流程，代表系统具备协作运行能力；只有真实发布并回填数据后，才能判断真实内容效果。
