# 多 Agent 角色与模型配置

## 1. 默认 Agent

MVP 阶段建议只使用 3 个 Agent。

| Agent | 默认职责 | 什么时候创建 |
| --- | --- | --- |
| 总控 Agent | 项目引导、派单、验收、文档、复盘、skill 判断 | 一开始就创建 |
| 素材 Agent | 素材整理、命名、分类和素材风险标记 | 飞书协作表建立后、首条任务开始前创建 |
| 内容生产 Agent | 参考素材分析、写作标准、任务单、初稿和合规自检 | 飞书协作表建立后、首条任务开始前创建 |

用户负责人工终审与是否发布的决定；总控负责将用户结论、发布链接和后续数据写入任务表并复盘。

## 2. 不建议一开始创建的 Agent

MVP 阶段不建议单独创建：

- Skill 沉淀 Agent
- 数据分析 Agent
- 发布 Agent
- 项目经理 Agent
- 质量检查 Agent

原因：终审与发布不应由独立 Agent 替用户作决定；合规自检前置在内容生产，终审与发布由用户负责，总控记录结果。

## 3. 模型选择原则

模型名称不绑定任何供应商。用户可以根据自己使用的平台替换。

| 工作类型 | 推荐模型等级 |
| --- | --- |
| 总控方案、冲突判断、复盘、skill 是否沉淀 | 强模型 |
| 首条视频模板、创意方向、复杂内容取舍 | 中高模型 |
| 普通素材整理、批量命名、字段回填 | 中低成本模型 |
| 明确脚本执行、批量生成版本 | 代码/执行型模型 |
| 合规争议、广告法、肖像、隐私判断 | 强模型 + 人工确认 |

## 4. Agent 推荐配置

| Agent | 默认模型 | 可降级场景 | 需升级场景 |
| --- | --- | --- | --- |
| 总控 Agent | 强模型 | 只查状态、整理格式 | 系统设计、流程冲突、复盘 |
| 素材 Agent | 中模型 | 批量重命名、台账整理 | 分类争议、素材风险不明确 |
| 内容生产 Agent | 中高模型 | 按明确任务单执行 | 参考标准提炼、创意判断、字幕风格、合规风险 |

## 5. 升级规则

如果 Agent 认为当前模型无法完成判断，不要硬做。

它应该在执行记录中写：

```text
当前任务难度升级，原因是：
1. ...
2. ...
建议由总控决定是否切换更强模型。
```

## 6. 成本控制原则

1. 强模型用在判断，不用在机械搬运。
2. 低成本模型用在批量整理，不用在复杂合规。
3. 视频生成前先任务单确认，减少反复渲染成本。
4. 每条视频都记录使用模型，便于后续测算 ROI。
