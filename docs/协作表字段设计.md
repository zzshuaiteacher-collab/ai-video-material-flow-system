# 协作表字段设计

MVP 阶段推荐只建 3 张表。

可以用飞书多维表格、Notion、Airtable、Google Sheet 或其他协作数据库实现。

## 1. 视频任务总表

用途：管理每条视频从选题到复盘的主状态。

推荐字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 视频编号 | 文本 | 例如 `V001` |
| 内容主题 | 文本 | 本条视频主题 |
| 内容类型 | 单选 | 菜品展示、产品展示、探店、口播、课程等 |
| 当前状态 | 单选 | 待生产、素材准备、参考分析中、剪辑中、合规自检、待人工终审、待发布、已发布、已归档、返工 |
| 当前负责人/Agent | 单选 | 总控、素材、内容生产、用户 |
| 参考素材与分析报告 | 长文本 | 链接、路径或分析摘要 |
| 写作/剪辑标准版本 | 文本 | 本任务引用的内容标准 |
| 采用模板版本 | 文本 | 例如 `T01@v1` |
| 采用音乐编号 | 文本 | 例如 `M01` |
| 生成视频路径 | 文本 | 本地或云端路径 |
| 内容合规自检结论 | 单选 | 未检查、通过、需修改、不通过 |
| 人工终审结论 | 单选 | 未审核、通过、需修改、不通过 |
| 发布状态 | 单选 | 未发布、待发布、已发布、已归档 |
| 发布链接 | 文本 | 真实或占位链接 |
| 发布链接类型 | 单选 | 真实发布、占位测试、竞品参考 |
| 发布时间 | 日期时间 | 真实发布时间 |
| 风险摘要 | 长文本 | 内容生产自检和用户发现的风险 |
| 最终人工决策说明 | 长文本 | 人工通过或驳回原因 |
| 数据回填状态 | 单选 | 未开始、待回填、已回填、无需回填 |
| 复盘结论 | 长文本 | 本轮复盘 |
| 下一步动作 | 长文本 | 当前需要做什么 |
| 备注 | 长文本 | 其他补充 |

## 2. 素材库表

用途：只管理待用素材、整理后素材、音乐及素材风险状态。参考素材分析、写作标准和任务采用模板归视频任务总表，由内容生产 Agent 负责。

推荐字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 资产编号 | 文本 | 例如 `A001` |
| 资产名称 | 文本 | 文件名或音乐名 |
| 资产类型 | 单选 | 原始视频、已整理素材、音乐、风险素材 |
| 分类 | 文本 | 例如制作、出餐、成品、环境、顾客 |
| 路径或链接 | 文本 | 本地路径或云端链接 |
| 可用状态 | 单选 | 可用、禁用、待确认、待分类 |
| 风险标签 | 多选 | 顾客正脸、订单、小票、支付码、版权、价格、地址等 |
| 适用场景 | 文本 | 例如制作过程、成品展示、环境、转场、背景音乐 |
| 备注 | 长文本 | 说明 |
| 最后更新 | 日期时间 | 更新时间 |

## 3. Agent 执行记录表

用途：记录每次 Agent 做了什么，方便总控追踪和复盘。

推荐字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 执行编号 | 文本 | 例如 `E001` |
| 关联视频编号 | 文本 | 对应视频任务 |
| 执行时间 | 日期时间 | 执行时间 |
| Agent | 单选 | 总控、素材、内容生产 |
| 任务类型 | 单选 | 初始化、素材整理、参考分析、内容生产、合规自检、归档记录、数据回填、复盘、skill建议 |
| 使用模型 | 文本 | 实际使用模型 |
| 执行状态 | 单选 | 待执行、执行中、已完成、需人工确认、失败 |
| 输入来源 | 长文本 | 读取了哪些文件或表 |
| 输出产物 | 长文本 | 产出了哪些文件或记录 |
| 问题记录 | 长文本 | 遇到的问题 |
| 是否建议skill化 | 单选 | 否、是、待评估 |
| skill化建议 | 长文本 | 建议沉淀的能力 |

## 4. 状态流转建议

```text
待生产
→ 素材准备
→ 参考分析中
→ 剪辑中
→ 合规自检
→ 待人工终审
→ 待发布
→ 已发布
→ 已归档
```

返工状态可以从任意阶段进入：

```text
返工
→ 回到对应责任 Agent
```

## 5. 发布链接类型规则

必须区分：

| 类型 | 能否做效果复盘 |
| --- | --- |
| 真实发布 | 可以 |
| 占位测试 | 不可以，只能做流程复盘 |
| 竞品参考 | 不可以，只能做拆解参考 |

## 6. 字段扩展原则

先不要加太多字段。

只有当满足以下任一条件时再扩展：

1. 同一个信息被反复写进备注。
2. 总控需要按该信息筛选。
3. 多个 Agent 都需要读取该信息。
4. 该信息会影响发布或合规决策。
